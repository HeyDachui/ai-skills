# 对话一致性回验 Skill

## 用途

本 Skill 用于帮助新 ChatGPT 对话、Codex 新线程或其他 AI 工具实例接手已有项目。

它解决的问题是：

新对话表面理解任务，但实际方向和用户真实意图不一致。

## 核心规则

未通过一致性验收前，不得进入生产。

## 推荐使用方式

在新对话中输入：

请启动《对话一致性回验 Skill》。

当前任务不是直接生产内容，也不是直接执行项目。

你的任务是先接手旧项目，并证明你是否真正理解项目边界、当前阶段、真实意图和不能做什么。

请先读取我提供的交接材料，判断材料是否足够，识别可能跑偏的风险。

如果材料不足，先用结构化访谈补齐。

如果材料足够，生成《新对话一致性验收包》，我会发给旧对话核验。

最高规则：未通过一致性验收前，不得进入生产。

## 标准流程

交接材料读取  
→ 材料充分性判断  
→ 风险识别  
→ 补充访谈  
→ 生成一致性验收包  
→ 旧对话回验  
→ 修正接手规则  
→ 用户确认  
→ 最小执行

## 适用对象

- ChatGPT 新对话
- Codex 新线程
- AI 工作流接手
- 项目交接
- 长对话卡顿后的新对话重启
- 内容生产项目
- 工程项目
- Obsidian 工作流
- 游戏 / 世界观项目

## 不适用情况

如果用户只是临时问一个简单问题，不涉及旧项目接手，不需要启动本 Skill。
