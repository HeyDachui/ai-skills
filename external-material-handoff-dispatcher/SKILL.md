# External Material Handoff Dispatcher

## Role

Translate collected external material into executable task packages for other threads without losing source, boundaries, or acceptance criteria.

## Delegate

Delegate repetitive low-dispute work:

- transcription
- OCR
- screenshots/keyframes
- contact sheets
- download status checks
- manifest checks
- visible comment extraction
- basic file inventory

Keep controller-level decisions with the intake controller:

- value judgment
- project routing
- category correction
- display grouping rules
- internalization eligibility
- final acceptance status

## Handoff Prompt Minimum

Include:

- receiving role name
- objective
- input paths
- output paths
- allowed actions
- forbidden actions
- UTF-8 requirement
- source/material type
- expected return format
- known gaps and uncertainty

## Acceptance Check

After a thread returns:

1. Verify output paths exist.
2. Check obvious encoding problems.
3. Confirm no raw files were moved/deleted/renamed unless authorized.
4. Confirm no Obsidian, PromptCopyPanel, global rule, or formal Skill write occurred unless authorized.
5. Summarize results for the user with clickable paths.
